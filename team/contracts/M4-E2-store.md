## Scope — implement `src/ask_chatgpt/store.py` (and finish `identity.resolve_conv_or_alias`)
Implement the §2.7 `Store` API and `ConversationPaths`. Avoid circular imports: `store` imports `identity`, never the reverse (put `resolve_conv_or_alias(value, store)` in `store.py`, or in `identity.py` guarded by `typing.TYPE_CHECKING`). Required behaviors (map to B1–B13):
- **B1 data-dir + layout:** `--data-dir`/explicit arg > `ASK_CHATGPT_DATA_DIR` > `~/.local/state/ask-chatgpt/`. `ensure_conversation(ref)` creates `conversations/<conversation-id>/` with `transcript.jsonl` parent, `raw-mapping.json` parent, `attachments/`, and `.gitignore` containing `attachments/`; never deletes existing files; project chats keyed by **chat id**, not project id.
- **B2 index:** `put_conversation_ref` creates/patches `index.json` (`schema_version:1`, `aliases`, `sessions`, `conversations`), preserving unrelated entries; `last_updated` only from `ConversationRef.updated_at` (RFC3339) else null — NEVER wall-clock. Index corruption is non-fatal (rebuildable).
- **B3 JSONL serialize/round-trip:** `upsert_turn` appends exactly ONE compact newline-terminated UTF-8 JSON object per record, all field names present incl. nulls, datetimes as RFC3339, tuples↔arrays, nested `ModelRef`/`AttachmentRef`/`CitationRef`; load reconstructs the immutable dataclasses semantically-equal. `upsert_many` appends many.
- **B4 read semantics:** `load_transcript(address, *, include_pending=False)` groups by `message_id`, keeps LAST valid record per id (last-writer-wins), hides pending local stubs (`message_id` starts `local:` AND `turn_index is None`) — whether superseded or not — by default; shows canonical superseding records; sorts by `(turn_index is None, turn_index, created_at or "", message_id)`. `include_pending=True` surfaces stubs WITHOUT giving them a turn index.
- **B5 torn line (DECISION 7):** one torn TRAILING line → `warnings.warn(StoreWarning(...))` + load prior valid records; mid-file invalid OR >1 trailing invalid line → raise `StoreError`.
- **B6 durability:** per-conversation advisory lock; flush+fsync complete lines; prior lines survive a simulated mid-append failure; concurrent appenders never byte-interleave a JSON object.
- **B7 atomic raw/index replace:** `write_raw_mapping_atomic(conversation_id, raw_tmp)` parses/validates candidate (top-level `mapping` + `current_node`) BEFORE `os.replace`; invalid candidate leaves old file intact; raw persistence is backend JSON ONLY — reject/strip any request-header wrapper (never write `authorization`/`cookie`/`oai-*` keys). Index replace also leaves the old complete index on failure.
- **B8 begin_send:** persists conversation/index entry then appends a UNIQUE `client_send_id` user stub `message_id="local:<client_send_id>"` with the prompt, requested model/tools, `turn_index=None`, `created_at=None`, `status="partial"`, `partial=true` (DECISION via E1 invariant). Returns that `TurnRecord`.
- **B9 commit_send:** appends a canonical user `TurnRecord` (backend/DOM id) with `supersedes_message_id="local:<client_send_id>"`; the original stub LINE remains in the file (append-only, not edited/deleted); default reads then show ONE confirmed user. No separate outbox file.
- **B10 record_partial:** appends an assistant record with supplied partial markdown, `status="partial"`/`"error"`, `partial=true`, linkage (`client_send_id`/`user_message_id`) when known, honest `capture_source`/`fidelity`, redacted stable `error` details (NO bearer/cookie/prompt canaries).
- **B11 render_markdown (DECISION 3):** visible default records only, in stable order; `## User`/`## Assistant`, blank line, literal `content_markdown`, one blank line between turns, exactly one trailing `\n`; no timestamps; never fetch attachments/citations; math/code/tables byte-preserved; hidden/superseded/pending content absent.
- **B12 payload helper (DECISION 3):** a public, independently-testable helper that emits payload to **stdout first** (flush) and ADDITIONALLY atomic-writes the same bytes to `--out` when given (`atomic_write_payload` = temp-in-same-dir + flush/fsync + rename); if the out write fails, stdout already received the payload, then raise `StoreError`. Handle str and bytes exactly (no `str(bytes)`, no newline normalization, no NUL truncation).
- **B13 attachment_path:** deterministic path under `conversations/<id>/attachments/`, sanitized against absolute paths/duplicates/`../` traversal.
- Finish **`resolve_conv_or_alias`/`Store.resolve_conversation`:** passthrough a `ConversationRef`; resolve bare ids and URLs statelessly (no index needed); consult `index.json` ONLY for non-address aliases/sessions; unknown non-address alias → `ConversationNotFoundError`. A corrupt index must not break URL/id resolution (B-A6).
- `history()`/`load_transcript` are **store-only — NEVER preflight CDP / open a browser** (acceptance + test-plan F1).
