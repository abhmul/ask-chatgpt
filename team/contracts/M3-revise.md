### G1 — BLOCKING (V2 GAP): repo/install isolation invariant missing from the safety section
The design's §7 safety summary omits a load-bearing safety invariant. Add to the §7 safety summary (and as a one-line caution in the §10 M4/M5 build sequence) the **repo/install isolation invariant**, transcribed from `team/charter.md` and `docs/REWRITE-SPEC.md §13`:
- **Never `git push`**; never merge to a published branch (outbound/irreversible — operator-reserved).
- **Never check out, commit to, merge into, fast-forward, or otherwise move the `stable` branch** (the installed tool is built from `stable`; moving/committing it mutates the tool another agent is using).
- **Never run `uv tool install` / `uv tool upgrade` / `… --reinstall`** (rebuilds/clobbers the installed tool). Development uses `uv run …` (project venv) only — `uv tool …` is forbidden.
State briefly WHY (it can mutate the operator's separately-installed running tool or cause irreversible outbound effects). These are constraints on the developers/agents building the tool and on any build/test workflow the design implies.

### W1 — (V2 WEAK): login/Cloudflare "poll read-only" not fully encoded
The design raises `HumanActionNeededError`/`HUMAN-ACTION-NEEDED` and never automates login (good), but does not encode the spec's **"poll read-only"** behavior after the stop. In §4.1 (and/or §5/§9 as appropriate), specify that on a login-wall/Cloudflare challenge the tool STOPS the action, logs `HUMAN-ACTION-NEEDED`, and then **polls read-only** (no sends, no login attempts, own diagnostic tab only) until the human resolves it or the caller gives up — never automating login. (`docs/REWRITE-SPEC.md §13`; `team/charter.md`.)

Also **define the referenced-but-unspecified seam types** the design names but never declares: `Transcript`, `SelectorMap`, `SendTimeouts`, `AttachmentSpec`, `PreflightResult`, `StatusReport` (give each a minimal typed definition — fields + types — so M4/M5 can implement without inventing them).

### M2 — MUST-FIX (V3): bound completion polling cost + state header lifetime
