## PHASE 1 — WS-idle CALIBRATION probe (pi worker; the gate) → `team/evidence/handoffs/M19/calibration.md`
Dispatch ONE pi probe worker. Transcribe the ENTIRE "REAL-SITE SAFETY for PHASE 1" block from `M19-common.md` VERBATIM into its prompt (especially the WS-PAYLOAD-LEAK rule). Its job:
1. **Preflight** `curl -s --max-time 5 http://127.0.0.1:9222/json/version` (200 + `webSocketDebuggerUrl`); else `BLOCKED:browser_down`.
2. **Write + run a probe script** (`uv run python` — project venv playwright; may reference the M18 approach + `src/ask_chatgpt/channels/cdp.py:595-599,635-639,892-902`):
   - `connect_over_cdp` → `context.new_page()` (OWN tab only) → `session = context.new_cdp_session(page)` → `session.send("Network.enable")`.
   - Subscribe to `Network.webSocketFrameReceived`, `Network.webSocketFrameSent`, `Network.webSocketCreated`. For each, record ONLY `{eventName, direction, monotonic_ts, opcode, payload_len_bytes}` — **NEVER `response.payloadData` / frame text** (WS-PAYLOAD-LEAK rule).
   - `page.goto("https://chatgpt.com/?temporary-chat=true")`; confirm temp-chat + composer, else `BLOCKED`.
   - Run these **≤4 prompts**, one at a time, **≥5 s apart**, each in the same temp chat (or a fresh temp chat per prompt): (a) `Reply with exactly: pong` (instant), (b) `Write one sentence about the ocean.` (short prose), (c) `List the numbers 1 to 20, one per line.` (chunked list), (d) `Write a 6-line Python function that returns the average of a list.` (code). For EACH: record `t0_send` at the send click; capture the WS frame arrival timeline until turn-end; detect DOM turn-end (`t_domdone` = stop button gone + assistant text non-empty + stable ~3 s).
   - If a rate modal appears at any point → `BLOCKED:rate_limit`, stop. Close ONLY your own tab; detach.
3. **Analyze (timing only):** per prompt, compute the sorted inter-frame gaps DURING the answer (between `t0_send` and `t_domdone`). Report **`max_mid_answer_gap`** across all prompts, and per-prompt `t_lastframe` vs `t_domdone`. Recommend **`N = ceil(max_mid_answer_gap) + margin`** (margin ≥ ~3 s). **Validate** the rule on the recorded data: simulate "fire when no WS frame for ≥ N AND DOM stable" — does it fire at/after each real turn-end (no clip) and NOT during any mid-answer gap (no false-early)? 
   - **Verdict `PASS`** iff a finite N exists that is (a) larger than every mid-answer gap (no false-early) and (b) small enough to be a usable completion latency (state the worst-case added latency = N). **`FAIL`** iff mid-answer gaps are so large/unbounded that no N cleanly separates "mid-answer pause" from "turn done" (then WS-idle alone is not viable — recommend pairing with a terminal-frame signal if one exists, or the lightweight-poll fallback).
4. **Write** `team/evidence/handoffs/M19/calibration.md`: the verdict, the per-prompt gap tables (timing/length only, NO payloads), `max_mid_answer_gap`, the recommended **N + margin + worst-case latency**, and the rule-validation result. Echo `STATUS: <verdict> N=<n>s` + the path.

**Manager:** block on the probe; read `calibration.md`; **re-derive N + PASS/FAIL yourself** from the recorded gaps (don't trust the worker's number — check the rule really has no false-early on the data). Gate per `M19-common.md`.
