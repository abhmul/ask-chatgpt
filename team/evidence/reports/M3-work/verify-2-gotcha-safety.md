## Part B — safety invariants

1. **CDP-attach only; no Playwright-launched browser — COVERED.** Requirement: `docs/REWRITE-SPEC.md §13` says “CDP-attach only; no Playwright-launched browser.” Design: `M3-detailed-design.md §2.9` says `CdpChannel` “attaches to an already-running browser only after `/json/version` succeeds” and “never calls Playwright launch.”

2. **No stealth / anti-detection — COVERED.** Requirement: `docs/REWRITE-SPEC.md §13` says “no stealth/anti-detection ever.” Design: `M3-detailed-design.md §7` safety summary includes “no stealth.”

3. **Domain allowlist enforced — COVERED.** Requirement: `docs/REWRITE-SPEC.md §13` requires a domain allowlist for chatgpt/openai/auth/oaiusercontent. Design: `M3-detailed-design.md §2.11` defines `allowlist.py`, `Allowlist.require_allowed_url`, and `DEFAULT_ALLOWED_HOST_SUFFIXES = ("chatgpt.com", ".chatgpt.com", "openai.com", ".openai.com", "oaiusercontent.com", ".oaiusercontent.com", "oaistatic.com", ".oaistatic.com")`; `§2.9` says navigation/fetches are allowlist-checked.

4. **Inspect only tool-opened tabs; never iterate `context.pages` — COVERED.** Requirement: `docs/REWRITE-SPEC.md §13` says “inspect only tabs the tool itself opens.” Design: `M3-detailed-design.md §2.9` says `CdpChannel` “creates only tool-owned targets” and “never iterates `context.pages`.” `§7` adds “The pool records only pages it created; mock tests should make `context.pages` unavailable.”

5. **Never quit browser; detach only / close own tabs only — COVERED.** Requirement: `docs/REWRITE-SPEC.md §13` says “never quit the browser (detach only).” Design: `M3-detailed-design.md §2.2` says `detach()` “disconnects/optionally closes only tool-owned tabs and never quits Chromium,” and `§7` says `close_all()` closes only pool pages then detaches “without quitting Chromium.”

6. **Preflight CDP before real leg; `CDP_UNREACHABLE` if down — COVERED.** Requirement: `team/charter.md §Shared-resource ceilings` requires “Preflight `curl -s --max-time 5 http://127.0.0.1:9222/json/version` before any real leg; if down → stop cleanly (`CDP_UNREACHABLE`) and escalate.” Design: `M3-detailed-design.md §2.2` says `attach()` performs preflight before real CDP use; `§2.9` gates attach on `/json/version`; `§9` defines `CDPUnreachableError` with code `CDP_UNREACHABLE` for “`/json/version` preflight fails/times out.”

7. **Login/Cloudflare challenge stop, log `HUMAN-ACTION-NEEDED`, poll read-only; login never automated — WEAK.** Requirement: `docs/REWRITE-SPEC.md §13` says “login/Cloudflare challenge → STOP, log `HUMAN-ACTION-NEEDED`, poll read-only; login is never automated.” Design covers stop/log/no-automation: `M3-detailed-design.md §4.1` says “Login/Cloudflare produces `HumanActionNeededError` with code `HUMAN-ACTION-NEEDED`; login is never automated,” and `§9` maps `HumanActionNeededError` to login wall/Cloudflare. It also keeps status checks on own diagnostic tabs in `§8`. Weakness: the design does not explicitly specify the required read-only polling loop/behavior after the stop; it only raises/stops. This is safer than automating login, but the contract phrase “poll read-only” is not fully encoded.

8. **Real-site legs operator-attended, never CI/cron/unattended — COVERED.** Requirement: `docs/REWRITE-SPEC.md §13` says real-site legs are “operator-attended (never CI/cron/unattended).” Design: `M3-detailed-design.md §1` says “Every real browser leg is operator-attended CDP attach,” and `§10 M5` labels real integration as “attended CDP.”

9. **Account human-paced; no programmatic spamming; no hard message cap — COVERED.** Requirement: `team/charter.md §Shared-resource ceilings` says “human-paced; no programmatic spamming; no hard message cap.” Design: `M3-detailed-design.md §7` says “There is no hard message cap,” uses politeness/backoff as safety knobs, and clarifies defaults are “assumptions to be measured, not a claimed account ceiling.” `§8` also says `loop --max-iterations` is “workflow control, not an account safety cap.”

10. **Authorization token + OAI headers never persisted/logged — COVERED.** Requirement: `team/contracts/M3-common-constraints.md §2` and M2 require required web-app headers but “Never persist/log the token.” Design: `M3-detailed-design.md §2.3` makes `HeaderBundle` `repr=False` and says values must never appear in “logs, exceptions, `raw-mapping.json`, `transcript.jsonl`, status reports, or test fixtures.” `§4.2` says header values are passed only as the fetch argument and “never returned or logged.” `§2.7` says `store.py` never receives auth/OAI headers.

11. **Never `git push`; never move/commit `stable`; never `uv tool install/upgrade/reinstall` — GAP.** Requirement is explicit in `docs/REWRITE-SPEC.md §13`: “Never `git push`; never move/commit `stable`; never `uv tool install/upgrade/reinstall`.” `team/charter.md §Repository, branch & installed-tool isolation` explains why this is load-bearing: the running installed tool changes if “the `stable` branch is moved or committed to” or someone runs “`uv tool install` / `uv tool upgrade` / `… --reinstall`,” then lists the absolute NEVER rules. The detailed design does not encode this invariant. Its only consolidated safety summary in `M3-detailed-design.md §7` lists “no Playwright-launched browser, no stealth, allowlist before navigation/fetch, own tabs only, never quit browser, CDP preflight before real legs, login/Cloudflare stop as `HUMAN-ACTION-NEEDED`, real legs attended, modest shared-browser use” and omits git/stable/uv-tool protections. The M4/M5 build sequence uses `uv run pytest` but does not state or enforce the forbidden `uv tool`/stable/push actions. Under the V2 verifier contract, this safety-invariant GAP blocks PASS.
