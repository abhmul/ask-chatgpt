## Test-strategy safety and falsifiability

| Proposed test | Safety/falsifiability check |
|---|---|
| No completion polling | Safe if done against `MockChannel`/unit code only. It must assert **no periodic full conversation GET**, not merely no `/stream_status`; the current path already has `backend_checks == 0` while `full_raw_fetches` increments (`tests/test_send_completion.py:727-752`, `src/ask_chatgpt/channels/mock.py:713-721`). It will fail on current main because `wait_for_completion` immediately calls `poll_backend_completion` (`src/ask_chatgpt/completion.py:141,159-162`). |
| One page-load per session | Safe offline. The test must assert reload count, not just open count: current tests already have an open-only assertion that passes (`tests/test_session_draft_loop.py:469`) and the two-iteration loop test only asserts a managed tab remains (`tests/test_session_draft_loop.py:511-575`). `MockChannel` records `open_tab` and `reload` (`src/ask_chatgpt/channels/mock.py:244-245,285-293,302-311`), so `reload == 0` is buildable and should fail on current main due to `src/ask_chatgpt/send.py:54-56`. |
| Rate governor | Safe offline with fake clock. Current `AdaptiveSendBudget` tests cover only in-memory send spacing (`tests/test_send_budget.py:8-80`), and the source governor is send-only (`src/ask_chatgpt/session.py:413`) with unwired `record_soft_signal` (`src/ask_chatgpt/session.py:194`; grep found no caller). A new all-request governor API test can fail current main by missing API or missing delay. |
| 429 distinct exit/backoff | Safe offline. Current completion non-200 raises `BackendCaptureShapeError` (`src/ask_chatgpt/completion.py:54-58`) and is swallowed in the wait loop (`src/ask_chatgpt/completion.py:160-164`); current capture non-2xx becomes `BackendCaptureShapeError` (`src/ask_chatgpt/capture.py:936-938`) then UI fallback (`src/ask_chatgpt/capture.py:358-360`), which by default raises clipboard `HumanActionNeededError` (`src/ask_chatgpt/capture.py:530-534`). Tests for `RATE_LIMITED:52` will fail current main. |
| Real-site request counter | Safe only if passive and attended. The synthesis’s design records paths/status/event names only and forbids bodies/header values/reloads/scrapes. It does not itself thrash; it is one human baseline message plus one short tool leg. |

One caveat: the “stream-close event” unit test is not buildable on today’s `BrowserChannel` protocol because there is no native-stream-finish method (`src/ask_chatgpt/channels/base.py:81-100`). That is expected implementation work; the test remains falsifiable if the mock event is merely an input and `wait_for_completion` must consume it without monkeypatching the answer.
